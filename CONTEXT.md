# 냉파 (cookmark)

냉장고 사진 한 장으로 재고를 파악하고, 사용자가 신뢰하는 저장 레시피와 매칭해 "오늘 뭐 해먹지"를 끝내는 제품의 도메인. 요리 가치사슬 ①재고 파악 → ②메뉴 발상 → ③레시피 실행 중 냉파는 ①→②를 담당하고, ③은 유튜브(저장 레시피)에 넘긴다.

## Language

**냉파**:
제품명. "냉장고 파먹기"의 준말 — 새로 장 보지 않고 지금 있는 재료로 해먹는 행위.
_Avoid_: cookmark(리포·코드 식별자로만 사용)

**질문 검증기**:
MVP의 정체. 완성 제품이 아니라 두 질문 — "사진 1장이 유튜브 검색보다 편한가", "저장 레시피가 실제 선택을 바꾸는가" — 에 답하기 위한 장치.
_Avoid_: 프로토타입(P1 스파이크와 혼동), 베타

**레시피 북**:
사용자가 직접 저장한, 출처 있는 레시피(유튜브·블로그 URL)의 모음. 냉파 차별화의 근거.
_Avoid_: 북마크, 즐겨찾기, 저장 목록

**재료 인식**:
냉장고 사진 1장에서 재료 후보 목록을 얻는 일. 결과는 항상 사용자가 체크박스로 수정할 수 있다.
_Avoid_: 스캔, 분석

**제안**:
인식된 재료와 레시피 북을 매칭해 내놓는 메뉴. 한 번에 최대 3개("오늘 할 3개") — 20개의 새로운 결정이 아니라 신뢰된 소수의 답.
_Avoid_: 추천 목록, 검색 결과

**제안 라벨**:
제안마다 붙는 실행 가능성 표시. "바로 가능" / "이것만 사면 가능" / "애매하지만 가능" 세 가지.

**부족 재료**:
제안된 메뉴에 필요하지만 인식된 재료에는 없는 재료.

**업로드 세션**:
30분 윈도로 묶인 사진 업로드 행위. 자발 사용 지표와 마찰 킬 기준이 공유하는 측정 단위.
_Avoid_: 업로드 횟수(재업로드 이중 계상 위험)

**폐기 베이스라인**:
파일럿 시작 전 구간에 기록한 "버려진 재료" 목록. 폐기 감소 판정의 비교 기준이며, 파일럿 시작과 동시에 마감된다.
