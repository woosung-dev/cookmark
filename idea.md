# cookmark

- 이름 / SaaS 한 줄

  냉파 — 냉장고 사진 한 장으로, 내가 믿는 레시피만 골라 "오늘 뭐 해먹지"를 끝내주는 앱이름 (cookmark)

- 누구의 / 어떤 한 문제 (웨지)

  장은 봐놨는데 안 해먹어서 채소·반찬거리를 자꾸 썩혀 버리는 1인 가구·맞벌이 가정.
  기존 앱은 출처 모를 generic 레시피만 줘서 신뢰가 안 가고 → 결국 안 만들고 → 또 버린다.

- 성공 기준: 데모 시나리오 1개

  냉장고 사진 업로드 → 재료 자동 인식(양파·계란·대파·두부) →
  "내 레시피 북"에 저장한 영상 레시피 중 만들 수 있는 3개를 매칭률 순으로 우선 노출,
  부족분은 LLM이 2개 보충 → 하나 선택 시 부족 재료·영상 링크 표시.

- 코어 기능 1개: 이것만 먼저

  사진 → 재료 인식 → "내 레시피 북 우선" 추천 랭킹.
  (영양 우선 추천 / 새로운 요리 제안은 2기로 미룸)

- SaaS 레이어 계획

  인증 : 카카오·구글 소셜 로그인 (레시피 북이 사용자별로 쌓여야 함)
  멀티테넌시 : 사용자별 레시피 북·냉장고 기록 분리 (user-scoped data)
  과금 : 무료(월 N회 사진 분석) / 프리미엄(무제한 + 영양 분석 + 무제한 저장)
  배포 : 모바일 우선 PWA → Next.js + Vercel/Cloudflare

- 차별화 포인트 1개

  추천이 "출처 있는, 내가 저장한 레시피" 우선.
  generic LLM 레시피가 아니라 내가 신뢰하는 유튜버·블로그 = 나만의 레시피 북 → 실제로 만들게 됨.

- 재사용할 1기 자산

  보안원칙 : 사용자 사진·식습관 데이터 최소수집·격리 원칙
  PLAN템플릿 : 본 계획서 양식
  위임검증SOP : LLM이 생성한 레시피/코드 검증 체크리스트

- 프리미엄(Freemium) 모델
  - 유료 과금(천장) 포인트
    - 레시피 북 확장: 기본 30개까지만 레시피 저장을 허용하고, 무제한 저장 및 폴더/태그 분류 기능을 프리미엄으로 뺍니다.
    - 영양/식단 분석 기능: 기획안에 2기로 미루신 ‘영양 우선 추천‘을 유료화하여, 다이어터나 헬스인 등 관리가 필요한 코어 타깃에게 과금합니다.
    - AI 레시피 무제한 튜닝: LLM이 부족한 재료를 보충해 주는 횟수를 제한하고, 이를 무제한으로 풀거나 ‘비건용으로 바꿔줘‘, ‘매운맛으로 어레인지 해줘’ 등의 심화 프롬프트를 유료로 제공합니다.
  - 제휴/어필리에이트(Affiliate) 모델
    - 직접 물건을 팔지 않고 수요자를 다른 서비스로 보내주며 중간 수수료를 받는 구조
      - 간단 ex) 쿠팡파트너스
      - 직접 마케팅 마켓컬리, 공동 구매 어플
