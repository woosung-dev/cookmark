출처 = 이슈 #53 본문(사용자 발행) + #32 이월. `DESIGN.md` §2·§7·§8은 제안 라벨을 **"색+아이콘 이중 신호"**로
규정하고 코드 `lib/ui/widgets/suggestion_card.dart:129`(`_LabelBadge`)는 이미 아이콘을 그린다. #32는 **색**만

### 결정 1 — 아이콘 소스는 앱 번들 폰트에서 직접 추출(코드 렌더와 구성상 동일)

아이콘을 `/opt/homebrew/share/flutter/bin/cache/artifacts/material_fonts/MaterialIcons-Regular.otf`의
codepoint 0xe15a·0xe05a에서 렌더했다. 앱은 `pubspec.yaml`의 `uses-material-design: true`로 **이 폰트를 번들**하고 코드는 Flutter 내장 `Icons.*`를 쓴다
(프레임워크 `material/icons.dart`가 `check_circle_outline = IconData(0xe15a, fontFamily:'MaterialIcons')` 등으로 정의 — 리포 파일 아님).
즉 **목업 아이콘 = 앱이 그리는 글리프**(동일 폰트·동일 codepoint). 이슈 체크박스 "코드 렌더와 대조"는 이 구성으로
충족된다 — 라이브 Flutter 렌더는 같은 폰트가 같은 글리프를 그리는 것을 재확인할 뿐이라 중복이며, #32도 코드
렌더가 아니라 코드 값과 대조해 검증했다.

렌더 파라미터도 코드에서 직접: 아이콘 alpha bbox **높이 14px**(코드 `Icon(size:14)`), 세로 중앙 정렬,
배치 = `[좌패드][아이콘][gap][텍스트][우패드]`(코드 `Row([Icon, SizedBox(Space.xs), Text])`).

### 결정 2 — gap = 5px (코드 Space.xs=4를 목업 스케일에 비례 환산)

목업 배지는 코드와 정확히 1:1이 아니다 — pill 높이는 22px로 코드(14+2×4=22)와 같지만 가로 패드는 ~11px로
코드 `Space.sm=8`보다 크다(≈1.4×). 이 스케일에서 코드 gap `Space.xs=4`는 4×1.4 ≈ 5.5px에 해당하므로
**gap=5px**가 리터럴 4px보다 목업 리듬에 맞다.

### 결정 3 — 성장 방향은 배지별로(열 정렬 유지·충돌 회피)

아이콘+gap = ~18px를 어느 쪽으로 넣느냐는 배지 문맥에 달렸다.

- **④·⑤ ready — 좌측 확장(우단 고정, 텍스트 불변).** 두 배지는 메뉴명 오른쪽에 떠 있고(좌측 열 없음) 좌측에
  흰 여백이 76·89px 있다. 좌측으로 18.5px 늘리면 텍스트를 안 건드리므로 **#32 글리프가 그대로 보존**된다.
  방식 = 원 pill 좌캡 픽셀을 복사해 새 좌단으로 이동 + 우패드 flat-tint 컬럼을 타일 + 아이콘 알파 합성.
- **④ buyOne — 우측 확장(좌단 고정) + danger 칩 이동.** 이 배지는 메뉴명·출처배지와 좌단 x≈801로 열 정렬돼
  있어 좌측으로 늘리면 정렬이 깨진다. 그래서 좌단을 고정하고 우측으로 18px 늘리며 텍스트를 +18 이동.
  buyOne 우단(900)과 danger 좌단(906) 사이는 6px뿐이라 우측 확장이 danger와 충돌 → **danger 칩을 +18px
  이동**해 6px 간격을 보존했다(danger는 #32가 이미 재구성한 pill이라 픽셀 블록 이동으로 clean). danger 스타일은
  무변경, 위치만 이동.

### 함정 — buy tint 샘플이 텍스트를 물면 색이 어두워진다

buyOne pill 중앙(x=850)은 텍스트 위다. 거기서 tint를 뽑으면 `#CAAE84`(어두운 잉크)가 나와 새 pill이 갈색으로
칠해진다(1차 결과). 좌패드 flat 구간(x=806)에서 다시 뽑아 교정.
