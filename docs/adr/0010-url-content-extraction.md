# 레시피 재료를 URL 내용에서 추출한다 — "제목만·아무것도 안 긁는다" 결정 역전

2026-07-19 사용자 결정으로 레시피 재료 추출을 **제목 추론에서 URL 내용 기반으로 승격한다**([#123](https://github.com/woosung-dev/cookmark/issues/123) · PR [#124](https://github.com/woosung-dev/cookmark/pull/124) main `994ce17`). 이는 스펙 #13과 `docs/tickets/13/context-notes.md`가 **법무 근거로 고정했던** "본문·자막을 긁지 않는다 — 제목만 LLM에 보낸다" 결정을 역전한다. ADR-0009가 인증 역전을 정직하게 기록한 것과 같은 취지로, 이 역전과 그 법적 포지션 변화를 여기 남긴다.

**무엇이 바뀌나 (구현은 #123에 랜딩)**

- **추출 사다리** — ①유튜브 URL이면 Gemini `file_uri`로 **영상 직독**(앞 20분) ②일반 http(s)면 서버가 페이지를 **fetch → JSON-LD `Recipe.recipeIngredient`(LLM 무호출) → 본문 텍스트 → LLM** ③결정적 단이 실패하면 다음 단으로, 최종 단 = 기존 제목 추론.
- **최종단 보장** — 결정적 단(분류·fetch·파싱)의 실패는 전부 제목 추론으로 강등해 저장 성공률이 현행 이상이다. LLM 자체 다운(`UpstreamLLMError`)만 502로 전파(add=미저장).

**역전되는 원래 결정과 그 근거**

- `context-notes.md`: *"본문·자막을 긁지 않는다 — 제목만 LLM에 보낸다. … 수익화·법적 리서치 #5의 YouTube 스크래핑 금지에 걸리지 않는다 — 아무것도 긁지 않는다."* → 이제 블로그 본문을 서버가 fetch·파싱하고, 유튜브 URL을 Gemini에 넘겨 영상을 읽는다.
- 스펙 #13 Out of scope: *"레시피 본문 스크래핑·저장 — URL·제목 수준만."* → fetch·파싱은 하되 **원본 본문은 저장하지 않는다**(재료 목록만 추출·저장). "저장"의 경계는 지킨다.

**법적 포지션 — 회색지대를 의식적으로 감수한다 (`docs/research/monetization-legal.md` 재검토 필요)**

- 법무 리서치는 *"B 단계에서 자동화 수집을 붙이는 순간 재검토 필수"*라 명시했고, 블로그 서버 fetch가 그 트리거다. 크롤링 3중 심사(저작권법 §93 DB제작자 권리 · 정보통신망법 §48① · 부정경쟁방지법 §2①(파))와 야놀자 v 여기어때(형사 무죄·**민사 10억 배상**)가 배경이다.
- **감수 근거** — (a) 사용자가 URL을 직접 제공하는 pull 방식(반복·체계적 크롤이 아님·야놀자형 "경미·공개"에 가깝다) (b) 추출·저장 대상은 **재료 목록**으로 리서치가 "저작권 비보호"로 확인한 것 (c) 원본 본문 미저장 (d) 파일럿 n=2 규모라 즉각 리스크가 낮다.
- **유튜브는 별도 회색지대** — 공식 YouTube API를 쓰지 않고 Gemini에 URL을 넘기는 방식이라 API 약관(30일 규정 등)의 직접 대상은 아니나, 이 해석 자체가 미검증이다.
- **미결로 남기는 것** — 야놀자형/사람인형 판정의 실제 법률 자문은 받지 않았다. 파일럿을 넘어 **실사용자·공개 배포로 가기 전 법률 확인을 의무**로 이월한다. `monetization-legal.md`의 "B 단계 이월 판단 항목"에 이 역전을 재검토 대상으로 표기했다.

**살아 있는 제약**

- **원본 저작물 본문 미저장** — 재료 목록만 얻고 페이지·자막 전문은 저장하지 않는다(§12 최소수집과 정합).
- **SSRF 가드** — 서버측 fetch는 `common/urlguard.py`(ADR 없는 #102 판정식의 공유화)로 최초 URL·리다이렉트 hop·스킴을 재검사한다.
- **자격증명 보호** — 유튜브 URL의 `user:pass@` userinfo는 Gemini 전송 전 제거한다.
- **seam 1개 유지** — AGENTS.md의 "LLM seam 1개" 무변경. `extract`가 url 파라미터와 추상 하위훅(영상·본문·제목)을 얻었을 뿐 경계는 하나다.

**재검토 트리거** — 파일럿 종료 후 공개 배포 결정 시 법률 자문(야놀자형/사람인형 판정) · 블로그 fetch가 특정 사이트의 robots/ToS 차단에 걸릴 때 · 공식 YouTube API 채택 시(그때 30일 갱신·메타데이터 원형 표시가 #123 리서치의 "B 단계 설계 제약 후보"에서 부활).
