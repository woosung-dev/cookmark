# 수동 수정 산식 — 모든 체크리스트 조작을 센다

P2 킬 기준("사진당 수동 수정 2주 연속 주 평균 5개 초과")의 산식을 "재료 체크리스트에서 사용자가 가한 모든 조작 각 1회"로 고정한다 — 체크 해제(오인식 제거), 재체크(low 항목 복원), 직접 추가(타이핑·칩), 뭉뚱그림 치환(1시퀀스 1회), 뭉뚱그림 오탐 복귀(휴리스틱이 잘못 분류한 점선 칩을 일반 항목으로 되돌리는 탭)를 전부 포함하고, low 환각을 그대로 무시한 것만 0이다. 킬 기준의 취지가 모델 오류율이 아니라 마찰 측정(유튜브 검색창 대비 손이 가는 정도)이므로, 사용자 손이 간 횟수 전부가 계측 대상이다. 파일럿 중 산식을 바꾸면 2주 데이터가 비교 불가가 되므로 이 정의는 파일럿 종료까지 불변이다.

## Considered Options

- **재체크 제외(모델 오류만 계측)** — low 항목이 실물에 있어 재체크하는 것은 "정정"이 아니라는 논리. 기각 — 마찰 측정이라는 취지에서 벗어나고, low를 남발하는 모델에 유리한 허점이 생긴다.
- **low 기본 체크 + 해제·추가만 계측** — 산식은 단순해지나 P1 실측(환각이 low/medium에 몰림)상 환각이 전부 체크되어 매칭이 오염된다. 기각.
- **오탐 복귀 제외(휴리스틱 오작동 정정은 마찰 아님)** — 오탐 복귀는 모델 인식이 아니라 클라이언트 뭉뚱그림 휴리스틱(ADR-0002)의 오작동을 정정하는 것이라는 논리. 기각 — 사용자에겐 원인 무관하게 탭은 마찰이고, 예외를 파면 재체크 제외와 같은 허점이 생긴다. 휴리스틱 소음은 산식에서 숨길 게 아니라 튜닝할 신호이며, 제외 산식은 `vagueDismiss` 별도 kind 로그로 분석 시 재산 가능하다.

## Consequences

- 이벤트 로그는 조작마다 유형(해제/재체크/추가/치환/오탐복귀)과 경로(타이핑/칩)를 분리 기록한다 — 분석 단계에서 재체크·오탐복귀 제외 산식으로 재산할 수 있는 원본 해상도를 남긴다.
- confidence 초기 상태(high·medium 체크, low 해제)가 산식과 한 몸이다 — 초기 상태를 바꾸면 산식 재검토가 필요하다.
- 뭉뚱그림 오탐 복귀 포함은 재확인 지도(#27) 티켓 #28에서 확정(2026-07-16) — ADR-0002가 보장한 "오탐은 탭 1회 복귀"가 만드는 마찰도 취지("손이 간 횟수 전부")대로 계측 대상이며, `vagueDismiss` 별도 kind 로그로 제외 산식 재산 가능성은 보존된다.
