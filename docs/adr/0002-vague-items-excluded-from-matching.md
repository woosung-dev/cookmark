# 뭉뚱그림 항목은 구체화 없이 매칭에 보내지 않는다

재료 인식이 내놓는 "반찬통"·"소스류" 같은 뭉뚱그림 항목은 재료 체크리스트 본문과 분리된 점선 칩으로 표시하고, 사용자가 인라인 치환으로 구체 재료("반찬통"→"멸치볶음, 김")로 바꾼 것만 매칭에 전송한다. 치환하지 않은 칩은 조용히 제외된다. "동의어 판정은 LLM이 한다"는 매칭 규칙이 있음에도 화면단에서 차단하는 이유 — "소스류"는 매칭 LLM이 활용할 정보가 0이면서 제안 근거를 오염시키는 노이즈이고, 치환 행위는 사용자의 실제 재고 지식을 데이터로 바꾸는 보너스이기 때문이다(P1 스파이크 실측: 뭉뚱그림은 flash-lite 계열의 상시 출력 패턴).

## Consequences

- 뭉뚱그림 판정은 클라이언트 휴리스틱(접미사 "~통/~류" 등 + low confidence 가중)이라 오탐이 가능하다. 오탐 항목은 칩 탭 한 번으로 일반 항목 복귀가 가능해야 한다.
- 치환은 수동 수정 1회로 계측된다(ADR-0003).
