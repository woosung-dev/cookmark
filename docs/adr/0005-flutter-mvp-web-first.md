# MVP를 Flutter로 구현한다 — 파일럿은 Web 빌드로 배포

승인된 디자인 문서는 Approach A의 스택으로 Next.js 단일 앱을 명시했으나, 2026-07-13 사용자 결정으로 MVP를 Flutter(Dart) 단일 코드베이스로 전환한다. 트레이드오프 — 파운더의 Flutter 학습 목표(파일럿 결과와 무관하게 회수)와 Go 이후 네이티브 앱 전환 시 코드베이스 연속성을 얻는 대신, 첫 Flutter 프로젝트의 학습곡선으로 파일럿 게이트(7/20) 이탈 위험을 감수한다. 파일럿 배포는 Web 빌드로 하여 G2 운영 규약(카톡 URL 온보딩·설치 0·즉시 수정 배포)을 그대로 유지한다.

> **⚠️ 배포 타깃 부분 역전 (2026-07-19, [ADR-0011](0011-native-android-pilot-target.md))** — "파일럿은 Web 빌드로 배포" 결정은 **네이티브 Android APK로 역전**됐다(지도 [#129](https://github.com/woosung-dev/cookmark/issues/129) 그릴링, 게이트 GO [#134](https://github.com/woosung-dev/cookmark/issues/134)). **Flutter 단일 코드베이스 결정은 살아 있다 — 역전 대상은 배포 타깃이지 스택이 아니다.** 아래 본문·Consequences는 2026-07-13 시점의 판단 기록이므로 그대로 두되(#31 재서명 관례), 배포 타깃에 관한 줄은 ADR-0011이 정본이다. 영향받는 줄에 인라인으로 역전 포인터를 단다.

## Considered Options

- **Next.js 유지(디자인 문서 원안)** — 검증 순도·속도 최우선. 기각 — 학습 목표를 회수하지 못하고, Flutter는 원안~디자인 문서 어디에서도 검토된 적 없는 신규 입력이었다.
- **Flutter 네이티브(설치 배포)** — 기각 — 배우자 기기 설치 마찰, 수정마다 재배포·재설치가 파일럿 요구와 충돌.
- **파일럿은 웹(Next.js), Flutter는 파일럿 기간 학습 병행** — 기각(사용자) — 냉파 코드에 Flutter를 즉시 적용하기를 원함.

## Consequences

- 게이트 규칙은 불변 — D0 = max(배포 완료, 폐기 베이스라인 7일 충족) 후 첫 저녁. 어느 쪽이든 늦으면 D0가 밀리고 폐기 베이스라인·요리 일지 구간만 길어진다.
  - **실측 (2026-07-16, #31 재서명)** — 배포 완료 7/16(PR #37) · 베이스라인 7/15 시작 → 7/21 충족. 따라서 **D0 = 2026-07-22 저녁**이다. 이 ADR을 쓸 때 적어둔 "7/20"은 베이스라인이 7/13에 시작한다는 전제로 계산한 값이었고, 실제 시작이 7/15라 두 날 밀렸다. **규칙이 아니라 입력이 바뀐 것이라 이 ADR의 결정은 그대로다** — 위 본문 "파일럿 게이트(7/20)"는 2026-07-13 시점의 판단 기록이므로 그대로 둔다.
- 네이티브 Android 빌드는 파일럿 이후 같은 코드베이스의 후순위 타깃(두 기기 모두 Android — APK 직접 전달 가능).
  - **역전 (2026-07-19, ADR-0011)** — "파일럿 이후"가 "파일럿 자체"로 당겨졌다. 네이티브 APK가 **파일럿의 배포 타깃**이다("두 기기 모두 Android — APK 직접 전달 가능"이라는 이 줄의 전제가 그 전환을 싸게 만들었다). 웹 폴백은 없다 — 못 맞추면 D0를 미룬다.
- 웹 전제 결정들(인앱 브라우저 경고·브라우저 로컬 영속·JSON 복사 백업·debug 쿼리 파라미터 측정 푸터)은 Web 빌드이므로 전부 유효.
  - **역전 (2026-07-19, ADR-0011)** — 이 넷은 네이티브 등가물로 이전됐다 — 인앱 브라우저 경고 소멸(#135) · 로컬 영속은 서명에 묶인 `SharedPreferences`(#132) · JSON 백업은 클립보드 유지(#136) · `?debug` 푸터는 네이티브에서 구조적 false라 **숨은 제스처**로 치환(#136·#143). 단 `web/`·조건부 임포트·배너 코드는 네이티브 관통 증명 전까지 **삭제하지 않는다**(ADR-0011 웹 폐기 시퀀싱).
- LLM 프록시(인식·매칭)는 Flutter 앱과 분리된 서버리스 함수로 유지 — API 키는 클라이언트에 두지 않는다.
