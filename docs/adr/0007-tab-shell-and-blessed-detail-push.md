# 코어 셸 = 하단 탭 바 2개 + 축복받은 제안 상세 push 1건 (ADR-0001 역전)

파일럿 목업(`docs/design/applied-app.jpeg`, 6화면) **풀 패리티**를 위해 ADR-0001의 "코어 루프는 단일 페이지·화면 2개·탭 바 없음·화면 전환 0회"를 되돌린다. 대신 **하단 탭 바**(메인 · 레시피 북)와 **카드 탭으로 여는 제안 상세 1개(명령형 push)**를 도입한다. 실행 방식은 **UI 먼저·백엔드 나중**(파운더 결정). ADR-0001이 지키려던 "앱 내비게이션 마찰 0"을 목업 시각 패리티와 맞바꾼 것이며, 파일럿 지표 해석에서 이 교환을 명시적으로 안고 간다.

## Considered Options

- **ADR-0001 유지(단일 페이지·전환 0회·헤더 링크 1개)** — 측정 순도(내비 마찰이 메커니즘 마찰에 섞이지 않음)는 최적이나, 파운더가 맞추라 한 목업과 구조가 어긋난다. 앱이 "휑하다"는 인상의 주원인이 이 절제였다. 기각.
- **풀 패리티(하단 탭 바 + 제안 상세 push)** — 목업 6화면을 그대로 재현. 측정 순도를 일부 내주는 대신 파운더가 원한 시각·구조를 얻는다. **채택.**

## Consequences

- **측정 순도 완화.** 앱 내비게이션 마찰이 더 이상 0이 아니다(탭 전환·상세 push). ADR-0001이 "남는 마찰 = 메커니즘 마찰"로 설계했던 순도가 약해지므로, n=2 파일럿 지표(특히 성공 지표 1 "사진 1장이 유튜브 검색보다 편한가")를 읽을 때 이 앱 내비 마찰을 상수로 두고 해석한다.
- **명령형 `Navigator.push`는 정확히 1건, `main_page.dart`에만.** 탭↔탭은 선언형(`root_shell.dart`의 `setState`), 제안 상세만 축복받은 단일 push다. `test/architecture/navigation_test.dart`가 이 불변식을 강제한다. 2번째 push를 추가하려면 먼저 `go_router` 도입을 재결정한다(#50 면제 유지 — go_router 미도입).
- **상세의 "이거 했어요"는 pop-with-result.** 상세는 결과와 함께 pop만 하고, 메인이 그 뒤에 `_markCooked`를 이어받아 실행취소 토스트를 **메인** ScaffoldMessenger에 띄운다(상세 위 오펀 방지). "영상 보기"는 로그+새 탭이라 pop 없이 주입 콜백으로 처리한다.
- **백엔드 이월 placeholder(프론트만으론 실값 불가) 명시:**
  - **매칭%** — 부족(대체 미해소) 수로 근사(`matchPercentOf`). 실 점수는 LLM 프록시가 반환해야 함.
  - **음식 사진** — `PhotoPlaceholder`(홍시-틴트). 실사진은 URL og:image → 웹 CORS 때문에 프록시 필요.
  - **담기(카트)** — 백엔드·모델 없음. 죽은 버튼 대신 **비상호작용 장식 칩**으로 둔다.
  - **있는 재료** — 저장 레시피의 재료 − 부족으로 **렌더타임 파생**(저장소 무변경). AI 제안·재료 빈 레시피면 섹션 생략.
  - **이어보기 타임스탬프(목업의 "· 2:14")** — 재개 위치 데이터가 없어 생략(정직). pill은 "영상 보기".
- **가짜 과금 크롬(레시피 북의 `N/30 저장됨`·무료 배지·프리미엄 무제한)** — 파일럿용 **UI 크롬일 뿐** 어떤 저장도 막지 않고 모델·저장소를 건드리지 않는다(파운더 요청).
- **인식 로딩 개수 미표기(정직).** 인식은 단일 async 호출이라 진행 중 재료 개수를 알 수 없다 — 목업의 "재료 N개 확인 중"은 표기하지 않는다. 진행바는 경과 시간 단계(early/mid/slow)에 연동된 실제 신호다.
- **온보딩 브랜드 히어로.** 홍시→차콜 그라디언트 위 흰 워드마크(≤34px, DESIGN §4 "40px+ 히어로 타이포" 상한 준수). 흰 텍스트는 브랜드 필(#E8552D, 대비 3.6)이 아니라 그라디언트 하단 차콜 위에 놓여 AA를 만족한다(DESIGN §2 브랜드=히어로 허용과 정합).
- **하드 제약은 그대로.** 이벤트 로깅·수동 수정 산식(ADR-0003)·`storage.dart`·`debug_footer.dart`·`loading_stage.dart` 무수정. `?debug` 푸터 포맷 보존. E2E는 검증의 정본으로 계속 그린(제안 상세는 +1 케이스로 잠갔다, 31→32).
- **문서 정합.** 코드·테스트가 이미 인용하던 "ADR-0007" dangling 참조를 이 문서가 닫는다. DESIGN.md §4·§7의 "탭 바 없음"·"하단 바 없음" 문구를 이 결정에 맞춰 갱신했다.
